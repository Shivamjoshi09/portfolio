Implementing Consent Mode v2 Without Breaking Your GA4 Data

Published June 2026  |  6 Min Read

Consent Mode v2 is now mandatory for advertisers serving European Union traffic to maintain core remarketing features and attribution in Google Ads and GA4. Setting it up correctly ensures compliance with the Digital Markets Act (DMA) while recovering gaps in user data.

Advanced parameters: ad_user_data & ad_personalization

Consent Mode v2 introduces two new key variables:

  • ad_user_data: Determines whether user data can be sent to Google for advertising purposes.
  • ad_personalization: Controls whether personalized advertising (remarketing audiences) can be activated for the user.

These exist alongside the traditional analytics_storage and ad_storage settings.

Basic vs. Advanced Consent Mode

Under the Basic implementation, no tags fire until consent is granted. Under the Advanced implementation, tags fire prior to consent but send cookie-less pings. Google then uses machine learning to model conversions for the non-consenting users.

Integrating GTM triggers with CMP interfaces like OneTrust or Cookiebot ensures that user options are captured immediately, safeguarding regulatory compliance without completely blinding analytics reports.
